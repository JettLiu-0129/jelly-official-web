# 翻译服务

## 使用建议

目前对于大多数翻译场景，使用大语言模型（LLM）进行翻译的效果，远比传统的机器翻译要好。如果您仅需要翻译文本，建议优先使用大模型服务。

Jelly 之所以支持翻译类型的服务，主要是为了支持词典类型的输出，像这样：

<img className="img-radius-sm" width={300} src="/dict-res.png" />

这个词典是带音标和发音的。内置工具中，也有使用大模型实现的词典，但是没有发音功能。

## 支持服务列表

目前 Jelly 内置仅支持系统翻译服务。您可以通过[安装插件](/plugin/add-service-plugin)来使用其他翻译服务。

### 系统翻译

气泡窗中的翻译固定使用系统翻译服务。

> 未来可能会支持比较稳定的大模型翻译服务。

系统翻译使用 Apple Translation API 实现，使用本地的神经网络引擎进行推理，速度极快，效果也还行。

:::tip
您可以把“AI 翻译”加入气泡窗或浮窗工具列表，在系统翻译质量不满足需求时，调用 AI 进行翻译。
:::

首次使用需要在 **设置 -> 语言与翻译** 中下载语言包。
